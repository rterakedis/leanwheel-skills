# Brief Quality Checklist

Rate each dimension: **strong / adequate / thin / broken**. Only write findings where they add information.

## 1. Decision-readiness
Could someone start a PRD from this without re-asking "what are we building"?

## 2. Honesty
Is "What Makes This Different" earned, or invented? Red flag: a differentiation claim any competitor could also make unchanged.

## 3. Problem specificity
Is the problem concrete (who, when, cost of not solving it) or generic boilerplate?

## 4. Scope discipline
Does Scope draw a real line, or is "Out" empty/theater?

## 5. Length discipline
Is this 1–2 pages, with overflow detail correctly routed to `brief-addendum.md` rather than bloating the brief?

## 6. Assumption hygiene
Every `[ASSUMPTION]` tag triaged — resolved or explicitly left open, never silently buried?

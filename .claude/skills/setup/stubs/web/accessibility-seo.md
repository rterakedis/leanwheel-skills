# Accessibility & SEO Rules

> Updated: 2026-06-12 — WCAG 2.2 AA floor
> Semantic HTML, accessibility mechanics, and per-page SEO requirements for all web surfaces.

---

## Semantic HTML First

The element IS the accessibility API. ARIA is a repair tool for the rare case where no semantic element exists — not a default.

| Need | Use | Never |
|---|---|---|
| Navigation action (goes somewhere) | `<a href>` | `<div onclick>`, `<span class="link">` |
| Action (does something) | `<button>` | `<a href="#">`, `<div role="button">` |
| Page regions | `<header> <nav> <main> <aside> <footer>` | `<div class="header">` etc. |
| Disclosure | `<details><summary>` | JS toggle on divs |
| Modal | `<dialog>` | div overlay + focus-trap library |
| Tabular data | `<table>` with `<th scope>` | div grid |
| Date/figures/quotes | `<time> <figure> <blockquote>` | styled spans |

Structure rules:
- Exactly one `<h1>` per page; heading levels never skip (h2 → h4 is a defect, not a style choice).
- Exactly one `<main>`; all content inside a landmark.
- Document language set: `<html lang="en">`.

## WCAG 2.2 AA Mechanics

These are the testable floor — each maps to an AC when a story builds UI:

- **Contrast:** text ≥ 4.5:1 (3:1 for large text); UI components and focus indicators ≥ 3:1. Check both light and dark schemes.
- **Focus:** every interactive element has a visible focus indicator (`:focus-visible`, never `outline: none` without replacement). Focus is never obscured by sticky headers/footers (WCAG 2.2 §2.4.11).
- **Keyboard:** every action reachable and operable by keyboard alone; no positive `tabindex`; DOM order = visual order.
- **Target size:** interactive targets ≥ 24×24 CSS px (2.2 §2.5.8); mobile tap targets ≥ 44×44.
- **Motion:** all non-essential animation gated behind `@media (prefers-reduced-motion: no-preference)`; nothing essential conveyed by motion or color alone.
- **Reflow:** page usable at 320px width and 200% zoom without horizontal scrolling.

## Forms

- Every input has a `<label for>` (placeholder is not a label).
- `autocomplete` attributes on identity/payment/address fields (2.2 §3.3.7 — don't make users re-enter known data).
- Errors: associated via `aria-describedby`, announced (live region or focus move), and specific ("Email must contain @", not "Invalid input").
- Required fields marked in text, not color alone.
- Never disable the submit button as validation — validate on submit and report.

---

## Images & Media

- Informative images: `alt` describes function/content in context. Decorative: `alt=""` (empty, present).
- `alt` never starts with "image of"/"picture of".
- Video: captions; audio: transcript.
- SVG icons inside buttons: `aria-hidden="true"` on the SVG, accessible name on the button.

## Per-Page SEO Requirements

Every page that ships must have — enforce via the layout/head component so a page *can't* omit them:

| Element | Rule |
|---|---|
| `<title>` | Unique per page, ≤ 60 chars, page-specific first |
| Meta description | Unique, 50–160 chars (build should fail on missing — schema-enforce in SSG frontmatter) |
| Canonical | `<link rel="canonical">` on every page, absolute URL |
| Open Graph | `og:title`, `og:description`, `og:image` (1200×630), `og:url`, `og:type` |
| Structured data | JSON-LD where content type matches (Article, BreadcrumbList, FAQ) — only for content actually on the page |
| `robots.txt` + XML sitemap | Generated by the SSG, referenced in robots.txt |
| RSS/Atom feed | Required for blog/news content; `<link rel="alternate">` in head |

Content site extras:
- Stable URLs — slug changes require a 301 redirect entry, never a silent break.
- 404 page with navigation back into the site.
- `hreflang` only when translations actually exist.

## Hard Rejections

| Banned | Replacement |
|---|---|
| `<div onclick>` / click handlers on non-interactive elements | `<button>` or `<a>` |
| `outline: none` without a visible replacement | `:focus-visible` styles |
| ARIA role duplicating a native element (`role="button"` on `<button>`) | Remove the redundant ARIA |
| Placeholder as the only label | `<label>` |
| Heading levels chosen for font size | Heading by hierarchy; size via CSS |
| Color as the only state/error signal | Text + icon + color |
| Missing `alt` attribute | `alt="..."` or explicit `alt=""` |

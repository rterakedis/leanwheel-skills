# Web Anti-Patterns & AI Guardrails

> Updated: 2026-06-12 — Baseline widely-available CSS / Astro 5+ / Hugo 0.146+
> Patterns frequently generated by AI from legacy training data. Each is a **hard rejection** in code review — do not approve PRs containing these.

---

## 1. Div Soup

**Symptom:** `<div class="header">`, `<div class="nav-link" onclick=...>`, `<span class="button">`.

**Why it's wrong:** Non-semantic markup is invisible to screen readers, keyboards, and search engines, and re-implements (badly) behavior the browser provides free.

```html
<!-- ❌ Banned -->
<div class="card-button" onclick="openCard()">View</div>

<!-- ✅ Replace with -->
<button type="button" class="card-button">View</button>
```

---

## 2. JavaScript for Native HTML/CSS Behavior

**Symptom:** A JS accordion, modal focus-trap library, scroll listener, tooltip library, or `IntersectionObserver` lazy-loader.

**Why it's wrong:** The platform ships all of these. JS versions are heavier, less accessible, and break without hydration.

| AI loves to write | Required replacement |
|---|---|
| Accordion component with state | `<details><summary>` |
| Modal + focus-trap package | `<dialog>` + `showModal()` |
| Tooltip/dropdown library | `popover` attribute |
| `window.addEventListener('scroll')` for sticky/shrink header | `position: sticky` (+ `animation-timeline: scroll()` for effects) |
| Smooth-scroll plugin | `scroll-behavior: smooth` |
| Carousel library for simple sliders | `scroll-snap-type` |
| JS lazy-load | `loading="lazy"` |

---

## 3. Inline Styles & Off-Token Values

**Symptom:** `style="margin-top: 13px"`, or stylesheet rules with raw hex/px values that exist in the token scale.

**Why it's wrong:** Every off-token value is design drift the next session will copy. Tokens in `docs/ux/DESIGN.md` are the contract.

```html
<!-- ❌ Banned -->
<p style="color: #2b6cb0; margin-top: 18px">

<!-- ✅ Replace with -->
<p class="lead">  /* .lead { color: var(--color-primary); margin-block-start: var(--space-4); } */
```

---

## 4. `!important` Escalation

**Symptom:** `!important` appearing to "win" against earlier styles.

**Why it's wrong:** It means specificity is already broken; each use forces the next override to escalate further. Fix the selector structure (flat, single-class) or layer order instead. Only acceptable use: overriding third-party inline styles you don't control — with a comment.

---

## 5. Hydrating Static Content (Astro)

**Symptom:** `client:load` on headers, footers, cards, or any component without user interaction; importing a React/Vue component to render static markup.

**Why it's wrong:** Ships a framework runtime to render HTML that never changes. This single pattern is the difference between a 0 KB and 200 KB page.

```astro
{/* ❌ Banned */}
<Header client:load />
<BlogCard client:visible post={post} />

{/* ✅ Replace with */}
<Header />
<BlogCard post={post} />
```

---

## 6. Bypassing the SSG Image Pipeline

**Symptom:** Raw `<img src="/images/...">` to `public/` (Astro) or `static/` (Hugo); images without `width`/`height`.

**Why it's wrong:** No resizing, no WebP, no `srcset`, and layout shift on every load. Use `astro:assets` `<Image />` or Hugo page resources + render hook.

---

## 7. Legacy Library Reflexes

**Symptom:** jQuery, Moment.js, Lodash, Bootstrap CSS, Font Awesome CDN tags — added for one function.

| Legacy reflex | Replacement |
|---|---|
| jQuery (`$()`) | `querySelector`, `fetch`, native DOM |
| Moment.js | `Intl.DateTimeFormat` / `Temporal` |
| Lodash for one helper | Native array/object methods |
| Bootstrap grid | CSS Grid + tokens |
| Icon-font CDN | Inline SVG (sprite or per-component) |
| CSS reset from 2012 | Small modern reset, `box-sizing: border-box` |

---

## 8. Layout by Absolute Positioning

**Symptom:** `position: absolute` + magic-number offsets to place layout elements; fixed pixel heights on text containers; floats for columns.

**Why it's wrong:** Breaks on the first content change, zoom level, or translation. Grid/flex handle every layout case; fixed heights truncate real content.

```css
/* ❌ Banned */
.sidebar { position: absolute; left: 0; top: 80px; width: 240px; }
.card { height: 180px; }

/* ✅ Replace with */
.layout { display: grid; grid-template-columns: 15rem 1fr; gap: var(--space-5); }
.card { min-height: 11rem; }
```

---

## 9. Hardcoded URLs and Paths (SSG)

**Symptom:** `https://mysite.com/blog/post/` or `/blog/post/` typed into templates or markdown.

**Why it's wrong:** Breaks on staging domains, subpath deploys, and slug changes. Hugo: `relref` / `relURL` / `.RelPermalink`. Astro: `Astro.site` + collection-derived paths.

---

## 10. `px` Font Sizes and Zoom-Hostile Units

**Symptom:** `font-size: 14px`, media queries in px tied to device names, `user-scalable=no`.

**Why it's wrong:** `px` font sizes ignore the user's browser font-size preference; disabling zoom is a WCAG failure. Type in `rem`, fluid scaling via `clamp()`, media/container queries in `rem`/`ch`.

---

## 11. Runtime Fetching of Build-Time Content

**Symptom:** `fetch('/api/posts.json')` or client-side markdown rendering on a static site; `useEffect` data loading for content that exists at build time.

**Why it's wrong:** The entire point of an SSG is that content is resolved at build. Runtime fetching adds spinners, layout shift, SEO loss, and a failure mode — for data that never changes between deploys. Use content collections (Astro) / `.GetPage`+ page resources (Hugo).

---

## 12. Remote Fonts and Third-Party CDN Calls

**Symptom:** `<link href="https://fonts.googleapis.com/...">`, `@import url(https://fonts...)`, `cdn.jsdelivr.net` / `unpkg.com` script or style tags.

**Why it's wrong:** Every third-party origin on the critical path costs a DNS + TLS round trip and caps the PageSpeed score; remote fonts also leak visitor IPs to the provider (a GDPR finding in several jurisdictions). The default is the **system font stack** (zero bytes). A brand-justified custom font is **self-hosted on the same origin** as subsetted woff2 through the site's asset pipeline — never fetched from a font CDN. Same rule for any script or stylesheet: if the site needs it, it ships from the site's own domain.

```html
<!-- ❌ Banned -->
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;700" rel="stylesheet">
<script src="https://cdn.jsdelivr.net/npm/somelib@3/dist/somelib.min.js"></script>

<!-- ✅ Replace with -->
<link rel="preload" href="/fonts/inter-var.woff2" as="font" type="font/woff2" crossorigin>
<!-- or better: no font files at all — font-family: system-ui, ... -->
```

---

## 13. Meta/SEO Afterthoughts

**Symptom:** Pages shipping with default or missing `<title>`, no meta description, no canonical, no OG image.

**Why it's wrong:** These can't be retrofitted by a crawler. Layout components must *require* title/description props; SSG frontmatter schemas must make description non-optional. A page that can build without them is the defect.
